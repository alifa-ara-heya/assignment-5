a. What is the Document Object Model (DOM)?

DOM (Document Object Model) is a programming interface for web documents, representing the structure of a webpage in a tree-like format. Each node in the tree represents a part of the document, such as elements, attributes, and text. The DOM allows scripts, like JavaScript, to dynamically access and modify the content, structure, and style of a document.

b. How do you select an element from the DOM?

We can select an element from the DOM using JavaScript methods like getElementById(), getElementsByClassName(), getElementsByTagName(), querySelector(), and querySelectorAll(). For example, document.getElementById('myId') selects the element with the ID "myId". The querySelector() method allows selecting the first matching element based on CSS selectors, such as document.querySelector('.myClass') for an element with the class "myClass". If we want to select multiple elements, querySelectorAll() returns a NodeList of all matching elements. Once selected, we can manipulate these elements, such as changing their content or styles./p>

c. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener is attached to a parent element to manage events for its child elements, even if those children are dynamically added later. Instead of adding individual event listeners to each child element, the parent captures events that bubble up from its descendants.

d. How do you manipulate an element's attributes and styles using the DOM?

We can manipulate an element's attributes using methods like setAttribute() and removeAttribute(), or directly access properties like element.id or element.src. To change styles, we can modify the style property, such as element.style.color = 'red' or element.style.display = 'none'. This allows us to dynamically update the appearance and behavior of elements on the page.
